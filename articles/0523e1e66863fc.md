# はじめに

[cargo-oxide](https://github.com/NVlabs/cuda-oxide)が公開されて、色々見ていたところ、そこで定義されていた`DisjointSlice`が気になったので調べてみました。


# cargo-oxideとは

[cargo-oxide](https://github.com/NVlabs/cuda-oxide)はRustからCUDAに変換するコンパイラです。

https://github.com/NVlabs/cuda-oxide/blob/faea3959950d3bb40efb6a29d03bfbc65725bae6/crates/rustc-codegen-cuda/examples/vecadd/src/main.rs#L35-L47

こんな感じでRustでCUDAカーネルを書くことができます。

RustのMIRからMLIRライクな[Pliron](https://github.com/pliron-org/pliron)をつかってLLVM IRにしてPTXにコンパイルします。

![pipeline](https://nvlabs.github.io/cuda-oxide/_images/compiler-pipeline.svg)

かなりよく動いていて、Rustほぼ全ての機能をそのままカーネルに持ち込めるのがありがたいです。

# CUDAとRustのSafety

先ほど`cargo-oxide`から引用した`vecadd`ですが、これをCUDAで書くと大体こんな感じです。

```cpp
__global__ void vecadd(int N, const float *a, const float *b, float *c)
{
    int i = blockDim.x * blockIdx.x + threadIdx.x;
    if(i < N) {
        c[i] = a[i] + b[i];
    }
}
```

これをRust視点でよく見てみると、`a`, `b`は読み込みだけなのでスレッド間で共有していいですが、cには書き込みできるのであらゆるスレッドが好きに`c`に書き込めてしまうと明確に **Shared XOR Mutable** に反してしまいます。
これを安全に解決するのが`DisjointSlice`です。

## ThreadIndexはスレッド間で共有してはいけない

CUDAにはShared Memoryというメモリがあり。そもメモリは同じブロック間で複数スレッドで共有できます。

`cargo-oxide`では`static`変数を使用するとSharedMemoryを使えます

https://github.com/NVlabs/cuda-oxide/blob/a6b70f0e363e2d6bb9e28ffd1e1eeac4455beafc/crates/rustc-codegen-cuda/examples/sharedmem/src/main.rs#L31-L51

SharedMemory経由でインデックスをやり取りされてしまうとまったく安全ではなくなるのでそれをライフタイムで制限しています

# DisjointSliceの使い方

もういちど

https://github.com/NVlabs/cuda-oxide/blob/faea3959950d3bb40efb6a29d03bfbc65725bae6/crates/rustc-codegen-cuda/examples/vecadd/src/main.rs#L35-L47

を見てみると`c`は`DisjointSlice`型で、`let idx = thread::index_1d();`でインデックスを得てから`c.get_mut(idx)`で`c`にアクセスしています。
セーフRustでは`DisjointSlice`に対して好きなインデックスを作ることはできず、必ずあらかじめ認められている方法でしかインデックスを作ることができません。
この例で言うと、`thread::index_1d`は必ずユニークなインデックスを返すので`c`にアクセスしても安全というわけです。

# `DisjointSlice`の中身

`DisjointSlice`は実質的に持っているのはポインタと長さだけ持っていて、`ThreadIndex`はインデックスを表す整数のみ持っています。
セーフティーはゼロコストで実装されています。

```rust
#[repr(C)]
pub struct DisjointSlice<'a, T, IndexSpace = Index1D> {
    ptr: *mut T,
    len: usize,
    _marker: PhantomData<&'a mut [T]>,
    _space: PhantomData<fn() -> IndexSpace>,
}

impl<'a, T, IndexSpace> DisjointSlice<'a, T, IndexSpace> {
    /*
        省略
    */
    #[inline]
    pub fn get_mut<'kernel>(&mut self, idx: ThreadIndex<'kernel, IndexSpace>) -> Option<&mut T> {
        let i = idx.get();
        if i < self.len {
            // SAFETY:
            // - Bounds check passed above.
            // - `idx` is a ThreadIndex derived from hardware built-in variables,
            //   guaranteeing a unique index per thread (no data races).
            // - The DisjointSlice was constructed with valid memory (from_raw_parts safety).
            Some(unsafe { &mut *self.ptr.add(i) })
        } else {
            None
        }
    }
    /*
        省略
    */
}
```

```rust
pub struct ThreadIndex<'kernel, IndexSpace = Index1D> {
    raw: usize,
    _kernel: PhantomData<fn(&'kernel mut ()) -> &'kernel mut ()>,
    _space: PhantomData<fn() -> IndexSpace>,
    _not_send_sync: PhantomData<*mut ()>,
}

impl<'kernel, IndexSpace> ThreadIndex<'kernel, IndexSpace> {
    /*
        省略
    */
    #[inline(always)]
    unsafe fn new(raw: usize, _scope: &'kernel KernelScope<'kernel>) -> Self {
        ThreadIndex {
            raw,
            _kernel: PhantomData,
            _space: PhantomData,
            _not_send_sync: PhantomData,
        }
    }
}
```

`&'kernel`はカーネルのスコープ全体の範囲のライフタイムで、`ThreadIndex`ではマーカーでその範囲を伸ばすことも短くすることもできないようにしています。
動かすにはまず`&'kernel`をどこかで作らなければならないので、`cargo-oxide`がproc-macroを使って自動的に各カーネルでライフタイムを持ったマーカーを先頭に挿入します。

https://github.com/NVlabs/cuda-oxide/blob/c3caeb8ea32af4b622054b7591275649f13d45b1/crates/cuda-macros/src/lib.rs#L1690-L1704

`ThreadIndex::new`にはこのマーカーが自動的に`cargo-oxide`によって引数に挿入されます。

# マーカー

`DisjointSlice`, `ThreadIndex`にあるマーカーを一つづつ見ていきます

```rust
#[repr(C)]
pub struct DisjointSlice<'a, T, IndexSpace = Index1D> {
    ptr: *mut T,
    len: usize,
    _marker: PhantomData<&'a mut [T]>, // 'a をここで使う。 mutなのでinvariant
    _space: PhantomData<fn() -> IndexSpace>, // ここでIndexSpaceを使う。`fn() -> IndexSpace`は型を所有せず使うことでIndexSpace由来の性質を持ち込まないためのイディオム
}
```

```rust
pub struct ThreadIndex<'kernel, IndexSpace = Index1D> {
    raw: usize,
    _kernel: PhantomData<fn(&'kernel mut ()) -> &'kernel mut ()>, // 'kernel をここで使う。 `fn(&'kernel mut ()) -> &'kernel mut ()`は&'kernelを伸ばすことも短くすることもできないように固定(invariant)するイディオム
    _space: PhantomData<fn() -> IndexSpace>, // IndexSpace使用
    _not_send_sync: PhantomData<*mut ()>, // !Send + !Syncを型に入れるイディオム。ThreadIndexはthreadごとに固有のはずなので!Send + !Syncを型に入れる必要がある。
}
```
