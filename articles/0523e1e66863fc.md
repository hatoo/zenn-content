# はじめに

[cargo-oxide](https://github.com/NVlabs/cuda-oxide)が公開されて、色々見ていたところ、そこで定義されていた`DisjointSlice`が気になったので調べてみました。


# cargo-oxideとは

[cargo-oxide](https://github.com/NVlabs/cuda-oxide)はRustからCUDAに変換するコンパイラです。

https://github.com/NVlabs/cuda-oxide/blob/faea3959950d3bb40efb6a29d03bfbc65725bae6/crates/rustc-codegen-cuda/examples/vecadd/src/main.rs#L35-L47

こんな感じでRustでCUDAカーネルを書くことができます。

RustのMIRからMLIRライクな[Pliron](https://github.com/pliron-org/pliron)をつかってLLVM IRにしてPTXにコンパイルします。

![pipeline](https://nvlabs.github.io/cuda-oxide/_images/compiler-pipeline.svg)

かなりよく動いていて、Rustほぼ全ての機能をそのままカーネルに持ち込めるのがありがたいです。

# CUDAとRustのSafety

先ほど`cargo-oxide`から引用した`vecadd`ですが、これをCUDAで書くと大体こんな感じです。

```cpp
__global__ void vecadd(int N, const float *a, const float *b, float *c)
{
    int i = blockDim.x * blockIdx.x + threadIdx.x;
    if(i < N) {
        c[i] = a[i] + b[i];
    }
}
```

これをRust視点でよく見てみると、`a`, `b`は読み込みだけなので範囲チェックさえちゃんとしていればスレッド間で共有していいですが、cには書き込みできるのであらゆるスレッドが好きに`c`に書き込めてしまうと明確に **Shared XOR Mutable** に反してしまいます。
これを安全に解決するのが`DisjointSlice`です。
